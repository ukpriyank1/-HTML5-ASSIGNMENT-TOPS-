Q-2. How to embed audio and video in a webpage?

The HTML5 audio Element is utilized

Recently released HTML5 <audio> element offers a standard method for including audio on web pages. While the audio component is more recent, it is compatible with the majority of current web browsers.

The sample below simply adds audio into an HTML5 document using one source specified by the src element and the browser's default controls.

<audio controls="controls" src="media/birds.mp3">
    Your browser does not support the HTML5 Audio element.
</audio>

Audio from several sources, with the defaults of the browser.

<audio controls="controls">
   <source src="media/birds.mp3" type="audio/mpeg">
   <source src="media/birds.ogg" type="audio/ogg">
   Your browser does not support the HTML5 Audio element.
</audio>

The 'ogg' track in the aforementioned example plays audio on Firefox, Opera, and Chrome but Internet Explorer and Safari require the'mp3' track.

Linking Audio Files

Your audio files can be linked to and played by checking the box next to them.

Consider the following illustration to illustrate how this functions in real life:

<a href="media/sea.mp3">Track 1</a> <a href="media/wind.mp3">Track 2</a>

Using the object Element

In HTML texts, the <object> element is used to embed several kinds of media assets. The specification indicates that an object may be any sort of media item, including audio, video, PDF files, Flash animations, and even photographs. Originally, this element was used to insert ActiveX controls.

The example code below is used to embed a basic audio file in a web page.

<object data="media/sea.mp3"></object> << data="media/sea.ogg"></object>

A section of a document is defined by the "section" element.

The HTML5 documentation provided by the W3C states that "a section is a thematic grouping of content, typically with a heading."

The home page of a website could be divided into sections for the introduction, content, and contact details.

Q-4 . Canvas and SVG tags

Both SVG and canvas have the ability to produce rich visuals in web browsers, yet they differ greatly from one another.

SVG

SVG is a vector-based graphics language that may be used to define images like text, circles, and boxes in XML-based image formats.

It is a language for XML-based graphical applications and 2D graphics. SVG can be shown by the majority of web browsers just like PNG, GIF, and JPG.

SVG images outperform bitmap graphics by a wide margin.

CANVAS

An HTML5 element called Canvas is used to draw visuals for the dynamic, scriptable rendering of bitmap pictures and 2D and 3D shapes on a website.

For the graphics to be drawn in Canvas, we need the script. It offers a blank graphic area that only a few JavaScript APIs (like Canvas 2D or WebGL) may draw on.
